Documentation

The design, build, and research record for Renata. Suggested reading order for someone new to the project, then a full index.

Start here (in order)

  1. ARCHITECTURE.md — the infrastructure design: dedicated source agents, isolation and health, honest freshness, and how serving stays read-only over precomputed tables.
  2. BUILD_PLAN.md — the original operating model: the three lanes (the machine, the manager, the owner), the six human gates, the data spine, the In-Play Score, and the go-to-market.
  3. PRODUCT_LAYER_PLAN.md — the six-phase build plan and every locked decision. This is the build order of record.
  4. PHASE2_API_BLUEPRINT.md — the read-only serving API: endpoints, the masking projection, pagination, and the test plan. Reflects what shipped.
  5. NETWORK_LAYER_PLAN.md — the native network layer: the broker directory, the "Competitor's Book," warm-intro paths, the entity-resolution model, and the connect-free / pay-to-unlock paywall.
  6. PROPOSAL_GENERATION.md — the assessment of whether Renata can auto-generate a broker proposal, the two-artifact model, and the data-gap map.

Reference and context

  • GTM_LINKEDIN_PLAYBOOK.md — where benefits brokers gather, the LinkedIn persona, and the compliant lead-gen mechanics.
  • EBI_PHASE2.md — the review record (three improvement rounds) on the Phase 2 API and the brand system, with applied and deferred items.
