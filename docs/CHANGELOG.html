Changelog

Notable changes to the Renata platform. Dates are the delivery date of the work.

2026-06 — Phases 0–3 implemented and verified

Phase 0 — Data spine

  • DOL Form 5500 ingest (main form + Schedule A), joined on ACK_ID, deriving renewal month and cost-per-head into companies + plans.
  • Synthetic fixture (data/2023/) so the full chain runs offline.

Phase 1 — Scoring and agents

  • The transparent In-Play Score (pipeline/score.py): rules-based, every point a reason code.
  • One dedicated SourceAgent per source with retry, heartbeat, and freshness SLA; the orchestrator and system_temperature() derive lead volume and feed health from real state.
  • The free Renewal Radar generator (pipeline/radar.py).

Hardening

  • Postgres-ready get_engine (dual direct/pooled connection roles, pool_pre_ping).
  • Column-drift detection fails loud and records source_health.ok=0 instead of loading nulls.
  • Idempotent signal writes (natural-key unique index + ON CONFLICT DO NOTHING).
  • Postgres-safe freshness_hours (datetime-vs-string and naive-timestamp handling).
  • Serving indexes for the Radar query.

Phase 2 — Read-only serving API

  • FastAPI layer in api/: GET /v1/radar (masked, ranked, keyset-paginated), GET /v1/accounts/{id} (402 for the locked tier), GET /v1/temperature (public, cached).
  • One server-side masking projection (identity hidden, real values shown); one entitlement() dependency as the auth seam; one-plan-per-company CTE to fix the one-to-many join.
  • Test suite (tests/api/).

Phase 3 — The easy button

  • Outreach drafting engine (api/drafting.py): template-first, Claude-optional, in Renata's voice; provenance discipline so numbers trace to data.
  • Draft endpoints: generate a 4-touch sequence, list, edit, approve. Gated to the paid tier; RENATA_DEV_UNLOCK=1 for local demo. No system send path (human-sent by design).
  • The in-app easy-button UI (frontend/renata_app.html).

Design and research (later phases)

  • Product-layer plan and locked decisions (docs/PRODUCT_LAYER_PLAN.md).
  • Network layer + paywall design (docs/NETWORK_LAYER_PLAN.md).
  • Auto-generated broker proposal assessment (docs/PROPOSAL_GENERATION.md).
  • Review record across the API and brand (docs/EBI_PHASE2.md).

Prototypes

  • End-to-end clickable wireframe (frontend/renata_workflow.html): the Radar dashboard, the account view, the drafted outreach, the generated switch-analysis proposal, the connect-CRM / upload step, and the close. Self-contained (no API), six-step navigation.

Packaging

  • Professional project scaffolding for handoff: root README, docs index, contributor guide, Makefile, pyproject.toml.
