CLAUDE.md — Renata

Context for working in this repo with Claude Code.

What this is

Renata is renewal intelligence for employee-benefits brokers. She reads public filings, fuses them into a per-company In-Play Score (which accounts are about to switch brokers), and drafts the outreach. The product persona — content, chatbot, in-app assistant — is also named Renata. Tagline: the oracle, and the easy button.

Architecture (data flows one direction)

dedicated source agents ──▶ signals + source_health ──▶ scoring (In-Play Score)
        │                                                      │
   DOL 5500 ingest ──▶ companies + plans ─────────────────────┘
                                                               ▼
                                          Renewal Radar generator + serving API
                                                               ▼
                                          frontend (landing, dashboard, app)

Each data source has its own dedicated agent (agents/) that owns access, cadence, retry, normalization, and health. One source failing never degrades the others. System freshness and lead temperature are computed from real state (system_temperature()), not faked — that powers the landing page's "last refreshed" and quality readouts.

Run it

pip install -r requirements.txt
cp .env.example .env                      # set DATABASE_URL (Supabase) or leave blank (SQLite)
python - <<'PY'                           # create schema
import sqlite3; sqlite3.connect("renata.db").executescript(open("schema.sql").read())
PY
python -m pipeline.run_phase0 --year 2023 --state TX   # ingest (uses sample fixture w/ --skip-fetch)
python -m pipeline.score --month 6                     # compute In-Play Scores
python -m agents.sources                               # run source agents + temperature
python -m pipeline.radar --state TX --out radar.html   # generate the Renewal Radar

A synthetic fixture in data/2023/ lets the whole chain run with --skip-fetch (no download).

Conventions

  • Scoring is transparent on purpose — every point maps to a plain-English reason code. Keep it that way; Renata has to explain herself.
  • Column drift lives in config.py — if a 5500 load warns about a missing column, fix the mapping there against DOL's File Layout, nowhere else.
  • Adding a data source = a new SourceAgent subclass in agents/sources.py + register it. Don't fold source logic into the pipeline.
  • The operating model has gates — pricing, compliance, spend, "publish as me," and a weekly green-light are human decisions; everything else runs autonomously.

Compliance guardrails (do not cross)

  • LinkedIn outreach is drafted, human-sent — never bulk-automated (TOS/bans).
  • 5500/WARN/Form D are public; SERFF rate-filing access varies by state (start with open ones).
  • Run Renata transparently as an AI analyst, paired with a real human face — never a fake person.
